Find\textquotesingle{}s the first directory entry for the provided path. 

All values recorded in p\+Dirent must be preserved to and between calls to \hyperlink{ff__dir_8c_a6ab102ccc88ac028ecbf7d349a6afa2f}{F\+F\+\_\+\+Find\+Next()}.

If F\+F\+\_\+\+F\+I\+N\+D\+A\+P\+I\+\_\+\+A\+L\+L\+O\+W\+\_\+\+W\+I\+L\+D\+C\+A\+R\+DS is defined, then path will have the following behaviour\+:

path = \char`\"{}\textbackslash{}\char`\"{} -\/ Open the root dir, and iterate through all items. path = \char`\"{}\textbackslash{}$\ast$.\+c\char`\"{} -\/ Open the root dir, showing only files matching $\ast$.c wildcard. path = \char`\"{}\textbackslash{}sub1\textbackslash{}newdir\char`\"{} -\/ Get the D\+I\+R\+E\+NT for the newdir directory in /sub1/ if one exists. path = \char`\"{}\textbackslash{}sub1\textbackslash{}newdir\textbackslash{}\char`\"{} -\/ Open the directory /sub1/newdir/ and iterate through all items. path = \char`\"{}\textbackslash{}sub1\textbackslash{}newdir\textbackslash{}$\ast$.\+c\char`\"{} -\/ Open the directory /sub1/newdir/ and iterate through all items matching the $\ast$.c wildcard.

\hyperlink{class_it}{It} is important to distinguish the differences in behaviour between opening a Find operation on a path like /sub1 and /sub1/. (/sub1 gets the sub1 dirent from the / dir, whereas /sub/ opens the sub1 dir).

Note, as compatible with other similar A\+P\+Is, Full\+F\+AT also accepts $\ast$ for the same behaviour as /sub1/.

For more up-\/to-\/date information please see the Full\+F\+AT wiki pages.
