◆ FF_GetPartitionBlockSize()

FF_T_SINT32 FF_GetPartitionBlockSize ( FF_IOMAN *  pIoman)

Returns the Block-size of a mounted Partition

The purpose of this function is to provide API access to information that might be useful in special cases. Like USB sticks that require a sector knocking sequence for security. After the sector knock, some secure USB sticks then present a different BlockSize.

参数
pIomanFF_IOMAN Object returned from FF_CreateIOMAN()
返回
The blocksize of the partition. A value less than 0 when an error occurs.
Any negative value can be cast to the FF_ERROR type.

FF_T_UINT8  PartitionNumber 
)

Mounts the Specified partition, the volume specified by the FF_IOMAN object provided.

The device drivers must adhere to the specification provided by FF_WRITE_BLOCKS and FF_READ_BLOCKS.

参数
pIomanFF_IOMAN object.
PartitionNumberThe primary partition number to be mounted. (0 - 3).
返回
0 on success.
FF_ERR_NULL_POINTER if a pIoman object wasn't provided.
FF_ERR_IOMAN_INVALID_PARTITION_NUM if the partition number is out of range.
FF_ERR_IOMAN_NO_MOUNTABLE_PARTITION if no partition was found.
FF_ERR_IOMAN_INVALID_FORMAT if the master boot record or partition boot block didn't provide sensible data.
FF_ERR_IOMAN_NOT_FAT_FORMATTED if the volume or partition couldn't be determined to be FAT. (
参见
ff_config.h)

◆ FF_RegisterBlkDevice()

FF_ERROR FF_RegisterBlkDevice ( FF_IOMAN *  pIoman,
FF_T_UINT16  BlkSize,
FF_WRITE_BLOCKS  fnWriteBlocks,
FF_READ_BLOCKS  fnReadBlocks,
void *  pParam 
)

Registers a device driver with FullFAT

The device drivers must adhere to the specification provided by FF_WRITE_BLOCKS and FF_READ_BLOCKS.

参数
pIomanFF_IOMAN object.
BlkSizeBlock Size that the driver deals in. (Minimum 512, larger values must be a multiple of 512).
fnWriteBlocksPointer to the Write Blocks to device function, as described by FF_WRITE_BLOCKS.
fnReadBlocksPointer to the Read Blocks from device function, as described by FF_READ_BLOCKS.
pParamPointer to a parameter for use in the functions.
返回
0 on success, FF_ERR_IOMAN_DEV_ALREADY_REGD if a device was already hooked, FF_ERR_IOMAN_NULL_POINTER if a pIoman object wasn't provided.
