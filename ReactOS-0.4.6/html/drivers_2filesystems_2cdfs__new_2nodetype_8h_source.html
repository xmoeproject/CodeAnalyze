58 // The following definitions are used to generate meaningful blue bugcheck
59 // screens. On a bugcheck the file system can output 4 ulongs of useful
60 // information. The first ulong will have encoded in it a source file id
61 // (in the high word) and the line number of the bugcheck (in the low word).
62 // The other values can be whatever the caller of the bugcheck routine deems
63 // necessary.
64 //
65 // Each individual file that calls bugcheck needs to have defined at the
66 // start of the file a constant called BugCheckFileId with one of the
67 // CDFS_BUG_CHECK_ values defined below and then use CdBugCheck to bugcheck
68 // the system.
69 //
